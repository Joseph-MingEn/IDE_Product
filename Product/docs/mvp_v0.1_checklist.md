# MVP v0.1 — 最小可行功能清單

本清單為 **`docs/mvp_definition.md`** 的**窄幅切片**：僅驗證「側邊欄對話 + 本機 Ollama + 目前檔案上下文」是否成立。  
**實作載體**：僅 **VS Code Extension**（`apps/vscode-extension`）。

---

## 範圍限制（v0.1 必須遵守）

| 項目 | v0.1 |
|------|------|
| 客戶端 | 僅 VS Code Extension，**不做** Tauri 桌面版 |
| LLM | **僅**連線本機 **Ollama**（HTTP，預設 `http://127.0.0.1:11434` 可設定） |
| NVIDIA NIM | **不做** |
| RAG | **不做**向量索引／embedding／Chroma／FAISS；**不**掃 whole codebase |
| Agent | **不做** tool loop、多步自主、MCP、終端、寫檔／patch |

---

## 最小可行功能（對應你的 6 點）

| # | 功能 | 驗收標準（可勾選） |
|---|------|---------------------|
| 1 | **側邊欄 Chat UI** | 於 VS Code Activity Bar 註冊圖示；點擊後在 **Side Bar** 顯示 Webview 或等同面板；版面含訊息區與輸入框。 |
| 2 | **輸入問題** | 使用者可在輸入框打字並送出；支援多行（可選）；送出後清空或保留可選，但不可無法送出。 |
| 3 | **呼叫本機 Ollama** | 使用選定模型呼叫 Ollama Chat API（例如 `POST /api/chat`）；模型名稱可設定（如 `qwen2.5-coder:7b`）；連線失敗時顯示可讀錯誤（拒連、404、timeout）。 |
| 4 | **顯示 AI 回覆** | 串流或非串流擇一即可；回覆顯示於訊息區且可捲動；區分使用者訊息與助理訊息（樣式或前綴）。 |
| 5 | **讀取目前開啟檔案內容** | 以 VS Code API 取得 **active editor** 的文件文字；若無開啟編輯器則提示「請先開啟檔案」；可選是否一併傳檔名／語言 id 以利模型解釋。 |
| 6 | **讓 AI 解釋目前檔案** | 在已讀取（5）之前提下，使用者可問「請解釋這個檔案」或等價；請求中**附上**目前檔案全文（或合理長度截斷策略並在 UI 註明）；模型回覆須針對該檔內容。 |

## 建議設定項（仍屬「最小」）

- Ollama **base URL**（預設本機）
- **Model** 名稱
- （可選）單次請求 **最大字元數**／截斷，避免極大檔塞爆上下文

---

## v0.1 驗收劇本（一條龍）

1. 在 VS Code 開啟任意原始碼檔並置於焦點。  
2. 開啟擴充側邊欄 Chat。  
3. 輸入「請用繁體中文簡短說明這個檔案在做什麼」並送出。  
4. 確認請求僅发往本機 Ollama；側邊欄出現合理回覆且與檔案內容相關。  
5. 關閉所有編輯器後再送出，應出現「請先開啟檔案」類提示而非靜默失敗。

完成以上即 **MVP v0.1 通過**。
